Binary Exploitation

Memory-safety bugs in compiled binaries. Buffer overflows, ROP chains, format strings, heap exploitation. Usually given a binary and an nc connection.

First commands to run
checksecchecksec --file=binary
fileConfirm architecture and type
stringsQuick scan for flags and hints
ROPgadgetFind usable gadgets for chains
Checksec output meaning
No NXStack executable, shellcode works
No PIEAddresses fixed, easier ROP
No canaryStack smashing without detection
Partial RELROGOT is writable, GOT overwrite possible
Common attack patterns
1
Buffer overflow: Use cyclic(100) to find offset, overwrite return address
2
ret2win: Jump directly to a win function by overwriting return address
3
ret2libc: Leak libc address, find version, calculate system() + /bin/sh
4
Format string: %x leaks, %n writes memory via printf bugs
5
ROP chain: Chain gadgets to set registers and call functions with NX enabled

Cross-referencesFind where the "wrong password" string is used

1

5
Complex validation? Use angr or z3 to solve constraints automatically

Cheatsheets

Universal challenge checklist, key term definitions, and category recognition guide.

Universal challenge checklist
1
Read title and description twice. The wording is always a hint.
2
Run file on every provided file. Never trust the extension.
3
Run strings <file> | grep -iE "flag|ctf|key|pass"
4
Run exiftool on any image, document, or PDF
5
Binary: run checksec then open in Ghidra
6
Text or encoded: paste into Multi-Decoder and CyberChef Magic
7
Network capture: Wireshark, follow every TCP and UDP stream
8
Stuck 15+ min: search "[category] [keyword from description] CTF writeup"
Crypto terms
PlaintextOriginal unencrypted message
CiphertextEncrypted or encoded output
EncodingReversible, no secret needed (Base64, Hex)
EncryptionReversible only with the correct key
HashingOne-way, not reversible by design
Web terms
XSSJS injected into another user's browser
CSRFForcing a logged-in browser to send requests
SSRFTricking server into making requests for you
IDORAccessing other users' data by changing an ID
RCERunning arbitrary commands on the server
Pwn / RE terms
StackMemory for local variables and return addresses
HeapMemory for dynamic allocation (malloc)
CanaryValue before return address, detects overflow
PIEPosition Independent Executable, randomized base
GadgetShort instruction sequence ending in ret, used in ROP
Forensics terms
Magic bytesFirst bytes identifying true file type
CarvingExtracting embedded files by signature
SteganographyHiding data inside another file
EXIFMetadata standard in image files
Memory dumpSnapshot of RAM at a point in time
Category recognition guide
CryptoGives ciphertext, public key, or n/e/c values. Mentions "encrypt", "cipher"
WebGives a URL to a live site or a Docker port to connect to
PwnGives a binary plus an nc host:port connection target
REGives a binary or APK with no remote connection, asks for "valid input" or "serial"
ForensicsGives a file (image, pcap, disk/memory dump) and asks "find the flag"
SteganographyGives a single image or audio file with no other context
OSINTGives a photo, username, or profile and asks "where / who / when"

Quick Tips

The habits that separate fast solves from hours of frustration.

When stuck
1
Re-read the challenge title and description, hints are hidden there
2
Run strings, file, exiftool first before anything complex
3
Paste into CyberChef Magic mode, it auto-detects most encodings
4
Search: "[category] [keyword] CTF writeup" for similar solved problems
5
Take a break. Fresh eyes solve more than frustration does.
PwnStars mindset
Question everythingNothing is what it first seems
Layer by layerClues are nested. Decode then decode again.
Read the hintsThe challenge description IS a hint
Document alwaysWrite down every tool and step tried
Post-solveRead writeups after solving to learn 10x more
CommunicateTell teammates what you're working on
Common rabbit hole traps
Wrong toolNo output after 2 tries? Move on, do not force it
Over-encodingBase64 of Base64 is possible. Keep layering if still gibberish.
Noise pixelsNot every pixel variation is LSB steganography
Wrong keyGibberish after decoding means wrong cipher or wrong key
